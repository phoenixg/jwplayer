加载平衡

  • 事先请阅读:   用 JW Player播放流式视频

    目的

    此向导会帮助你了解 JW Player所支持的加载平衡类型.

    介绍

    加载平衡（Loadbalancing） 是一项允许分配资源以共同协作的技术, 以应对计算机承受的负荷. 在视频分发的领域内, 使用它来分开加载的视频,以便确保视频服务的质量 .

    内容

    • SMIL 文件
    • HTTP 加载平衡
    • RTMP 加载平衡

    SMIL 文件

    SMIL (Synchronized Multimedia Integration Language) 文件是一个多媒体演示的文件类型. 为了帮助加载平衡, SMIL 文件会定义很多个位置, 每个位置都定位到一个服务器, 为每一个视频文件这么做. 播放器会随机地选择使用哪个位置上的视频, 这样实现平衡 “加载.”

    在 JW Player中,只有 RTMP MediaProvider 支持通过 SMIL 文件进行加载平衡. 要使用一个 SMIL 文件, 只需简单地设置 FlashVar 指向SMIL文件的URL , 然后设置 RTMP.加载平衡的 FlashVar 为true即可.

    HTTP

    DNS 加载平衡

    域名系统 (DNS) 会对域名或 URL进行翻译, 翻译成一个 IP 地址,对应到一个物理服务器. (广泛使用的类比是将 DNS 比作电话簿.) 因为 DNS 允许每一个域名有多个列表, 所以你可以在很多的机器间实现加载平衡,只需在同一个 DNS 条目内添加上所有的机器即可.

    这一过程会由 Flash 播放器自行处理, 但你需要配置一下你的 DNS 条目内包含所有的服务器.

    重定向

    服务器是这样处理重定向的,得到一个文件请求,然后给你一个新地址. 正因重定向可以指向任何服务器,所以这成为了在很多个服务器间高效的实现加载平衡的方法.

    这一过程会由Flash 播放器自行处理, 但你需要配置你的服务器来正确地进行重定向.

    RTMP

    RTMP 加载平衡的配置比 HTTP 加载平衡的配置更具挑战性,因为它需要专门完成. 注意比特率切换功能通常都是在 RTMP 服务器上完成的,所以你不能使用 DNS 加载平衡 和 RTMP的结合.

    FCSubscribe

    FCSubscribe是运行在RTMP服务器上的自定义的 ActionScript 代码. 播放器会嗅探主RTMP服务器上的此脚本 (通过 "connection.call" 命令) 并得到边缘RTMP服务器的 URL .

    对于 FCSubscribe 的支持是有限的—如果需要这个功能请保持使用 JW4版本.

    重定向

    最近, RTMP 开始实现原生对重定向的支持了. 这个公司使用的是和 FCSubscribe一样的脚本层, 但内置进了 FMS3 和 Wowza 1.7+ 服务器上.
