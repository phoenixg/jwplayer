JW Player支持的格式

本页列出了所有JW Player支持的媒体文件、流式文件和播放列表类型的格式。

单一文件

播放器总是按照文件扩展名来识别是哪种文件（即封装容器而非编码）。 如果没有找到合适的扩展名，播放器会假定你想加载一个播放列表！所以，你要设置 provider flashvar, 比如. provider=video.

  • 对于FLV和MP4视频，以及AAC音频(provider=video).
  • 对于MP3音频(provider=sound).
  • 对于JPG、GIF、PNG图片 (provider=image).

常见问题:

  • 尽管播放器也能加载swf文件，但不鼓励使用。因为播放器无法读取swf文件的时长和分辨率信息，也不能控制音量或回放。
  • 如果你没能在MP4文件下载完成前寻找到，那么会出现所谓MOOV atom (包含了寻找信息) ，它位于视频的最后。可以试试看 this little application 来解析你的视频并修复它.
  • 如果你的MP3文件回放速度过快或过慢，那就意味着该文件采用了动态比特率VBR进行了编码，或者采用了不支持的采样率（比如48Khz）。请改成恒定比特率CBR编码，并使用44Khz以下采样率。 4.x版本的播放器比3.x版本支持的格式更少 (因为AVM2 方面的bug).
  • 如果进度条没有跟你的flv文件一起走，或者如果你的视频分辨率显示错了，那就意味着你的flv文件的metadata错了，也就是你的flv文件缺少了metadata信息。可以用这个小工具进行修复：www.buraks.com/flvmdi/.

更多有关格式支持的技术信息见： this blogpost 作者Tinic Uro是flash插件的开发人之一. 此外还有一篇很棒的开源flash文件格式文章 supported Flash file formats.

Youtube视频

播放器内置了对Youtube视频播放的支持(provider=youtube). 只需把 file flashvar 设置成youtube视频的 URL 即可 (比如 http://www.youtube.com/watch?v=WuQnd3d9IuA).

播放器会使用官方的 Youtube API . 该 API通过独立的 yt.swf 文件进行获取. 为了能正常播放youtube文件, 你必须把 yt.swf 和 player.swf 放在服务器的同一个目录下.

不幸的是，目前还没有办法通过API播放youtube的高清视频.

HTTP Streaming / HTTP 流式文件

FLV和MP4视频都能使用所谓的“HTTP伪流式服务器”（HTTP pseudostreaming servers）进行播放. 这些服务器允许视频观看者跳转到视频尚未下载的部分进行观看。可以通过设置flashvar provider=http 来启用HTTP streaming . 以下是一些流式服务器的例子:

  • Apache, Lighttpd, IIS and NginX的 H264 streaming module（H264 流式模块）
  • Lighttpdmod_flv_streaming module（mod flv流式模块）.
  • PHP/ASP 脚本，比如 XmooV PHP.
  • 内容递送网络，比如 Bitgravity, Edgecast 或Limelight.

在provider=http flashvar的后面, 还有两个flashvars 可以用来配置 HTTP streaming:

  • 默认的， 提供给服务器起始点的querystring 参数被称作start. (比如 http://server.com/video.flv?start=3489512). 对于 4.6版本, 该参数可通过 http.startparam 这个flashvar进行配置。比如, Edgecast CDN使用 ec_seek, Bitgravity使用 starttime ，而 Limelight 使用 fs 作为伪流式（ pseudostreaming）的起始参数.
  • 如果你使用 PHP/ASP/* 脚本来实现伪流式（pseudostreaming）, 你就必须在脚本中使用 streamer flashvar. 不能直接用起始参数调用文件, 播放器会调用脚本, 要用文件和起始位置作为参数 (比如 http://server.com/streamscript.php?file=video.flv&start=3489512).

RTMP Streaming / RTMP 流式文件

FLV、MP4视频以及MP3、AAV音频也可以用 RTMP Servers 做流式, 这时要用 provider=rtmp flashvar. 此外，, RTMP 实时流也可以在播放器中播放. 播放器支持以下几个 RTMP 服务器:

  • Wowza Media Server
  • Flash Media Server
  • Red5 server

除了 file 和 provider=rtmp 这两个flashvar外, RTMP 流式通常还使用第三个 flashvar streamer, 它指定 RTMP服务器的位置 (比如. streamer=rtmp://edge.sercer.com/application).

The player will figure out by itself if it needs to strip the .flv extension off the file, or if it needs to prepend a mp4: or mp3: string to load the file. In 5.0, this behaviour can be skipped by setting the rtmp.prepend=false flashvar.

Most frequently used RTMP functionality (such as 302 redirects, the getstreamlength call and Wowza's secureToken) is built into the player. In addition to this, the player supports loadbalancing with XML files. This is used by e.g. the Highwinds and VDO-X CDNs. Instead of loading the video, the RTMP model presumes that the file flashvar is actually an intermediate SMIL file, which is then loaded and parsed to retrieve the actual RTMP streamer and file. In version 5.0 of the player, turn this on by adding the flashvar rtmp.loadbalance=true. Here's an example of such a SMIL file, in case you're interested in building this into your own CDN: 

<smil> 
    <head> 
        <meta base="rtmp://68.16.184.155/n9f8m2x7/_definst_"/> 
    </head> 
    <body> 
        <video src="fms/video?doppl=164314e15f9563d3&dopsig=9bfcdaefc371a52fe049bf87c03b7776"/> 
    </body> 
</smil>

A special RTMP case is the live broadcast through the Akamai or Limelight networks. These CDNs use a stream subscription mechanism for load balancing. This is supported by the player, by setting the rtmp.subscribe flashvar to true. The regular file and streamer flashvars are still needed. This parameter can also be parsed as a playlist variable, to allow a playlist with videos from different streaming services.

XML 播放列表

首先, 注意播放列表的XML文件遵循flash的 跨域名安全限制 . 这意味着同一域名上的播放器无法加载来自另一个域名的播放列表. 这个限制可以通过在服务器放置一个 crossdomain.xml 文件来解决. 也就是说, 支持以下的播放列表格式:

  • ASX feeds (example)
  • ATOM feeds with Media extensions (example)
  • RSS feeds with iTunes (example) extensions and Media extensions (example)
  • XSPF feeds (example)

以下是每种格式的所有标签,以及与之对应的 flashvars概览:

所有l media:标签可以被嵌入进 media:group 元素中t. 一个 media:content 元素也可以作为一个容器. 此外, iTunes 和 MediaRSS 标签可以被混合成一个 RSS feed. 播放器会为每个属性选取最后匹配的元素.

比特率切换

5.1版本的播放器开始支持比特率切换, 这有利于你针对每个访问者进行优化选择. 可以通过一个XML播放列表文件加载不同质量水平的视频启用比特率切换.不同质量水平的信息必须包含在 media:content 元素中，并且要定义 bitrate, url 和 width .下面是这样一个feed的例子: 

<rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" >
  <channel>
    <item>
      <title>Progressive download</title>
      <link>http://www.bigbuckbunny.org/</link>
      <description>Big Buck Bunny is a short animated film by the Blender Institute.</description>
      <media:group>
        <media:content bitrate="1500" url="http://bitsontherun.com/videos/Qvxp3Jnv-486.mp4"  width="1080" />
        <media:content bitrate="1100" url="http://bitsontherun.com/videos/Qvxp3Jnv-485.mp4" width="720"/>
        <media:content bitrate="700" url="http://bitsontherun.com/videos/Qvxp3Jnv-484.mp4" width="480" />
        <media:content bitrate="400" url="http://bitsontherun.com/videos/Qvxp3Jnv-483.mp4" width="320" />
      </media:group>
    </item>
  </channel>
</rss>

media:content 元素来实现比特率切换的作用是确保该 feed 也能同其他 RSS阅读器和媒体播放器 (比如 5.0 JW Player)兼容. 推荐根据质量对流进行排序, 质量最好的排在最前面. 这样大多数播放器会优先选择这一个视频.

media:group 元素是可选的,但能使feed看起来更加结构化一些.

JWPlayer 命名空间

为了给所有feed格式启用JW player的所有文件属性, 4.4版本的播放器引入了jwplayer 命名空间. 只要在feed里插入它, feed格式本身不支持的文件属性 (比如RSS feed不支持provider或duration) 就可以得到支持. 上表所列举的所有flashvars都可以插入. 下面是一个例子: 

<rss version="2.0" xmlns:jwplayer="http://developer.longtailvideo.com/trac/wiki/FlashFormats">
  <channel>
    <title>Example RSS feed with jwplayer extensions</title>
    <item>
      <title>FLV Video</title>
      <link>http://www.bigbuckbunny.org/</link>
      <description>Big Buck Bunny is a short animated film by the Blender Institute, part of the Blender Foundation.</description>
      <enclosure url="../../testing/files/bunny.flv" type="video/x-flv" length="1192846" />
      <jwplayer:author>the Peach Open Movie Project</jwplayer:author>
      <jwplayer:provider>http</jwplayer:provider>
      <jwplayer:duration>34</jwplayer:duration>
    </item>
  </channel>
</rss>

注意最上面的标签, 它用 xmlns 属性定义了命名空间.

你可以把 jwplayer 元素和feed的常规元素，以及其他扩展 (mrss/itunes)的元素混合在一起.如果多种元素适合同一个属性, 元素的优先级是:

  • feed格式自身的元素 (比如RSS的enclosure元素)得到最低的优先级.
  • itunes 命名空间的元素优先级排名第三.
  • media命名空间的元素(比如 media:content) 优先级排名第二 .
  • jwplayer 命名空间的元素优先级总是最高.

该功能让你给JW player设置一个特殊的版本或流 provider , 于此同时其他feed使用默认的东西.
