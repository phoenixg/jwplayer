JW Player: Media Providers ¶

Media Providers 让播放器能够在不同平台上播放不同类型的媒体. 设置 provider 属性 (以前在版本4中称作 type) 通过 flashvars 或 playlist 指示播放器使用相应的 Media Provider.

内置的Media Providers ¶

播放器包含以下几个内置的 Media Providers:

  • video: 指渐进下载式的 FLV / MP4 视频. (请注意, AAC 音频要求使用 video Media Provider).
  • http: FLV/MP4视频当作http pseudo-streaming伪流式视频来播放.
  • rtmp: FLV/MP4/MP3 文件从一个 RTMP 服务器 播放
  • sound: 渐进下载式的 MP3 文件.
  • image: GIF/JPG/PNG 图片.
  • youtube: 来自 YouTube的视频.

外部加载的 Media Providers ¶

除了以上列出的几个 Media Providers之外, 还可以加载外部的Media Providers. 有两种外部的 Providers 可以被加载; 如果 provider 包含了 指向一个.swf文件的URL 包含Media Provider,那么就会被加载. 如果 provider 包含了 host在LongTail 上的 Provider, 那么就会被加载.

例子 ¶

以下例子演示了使用 Media Providers 的不同方法

单一播放列表项目 ¶

<config>
   <file>myvideo.flv</file>
   <provider>video</provider>
</config>

多重播放列表项目 ¶

<rss version="2.0" 
	xmlns:media="http://search.yahoo.com/mrss/" 
	xmlns:jwplayer="http://developer.longtailvideo.com/trac/wiki/FlashFormats">
	<channel>
		<title>Example media RSS playlist</title>
		<item>
			<title>FLV Video</title>
			<media:content url="video.flv" />
			<jwplayer:provider>video</jwplayer:provider>
		</item>
		<item>
			<title>HTTP Streamed FLV</title>
			<media:content url="video.flv" />
			<jwplayer:provider>http</jwplayer:provider>
			<jwplayer:streamer>http://myserver.com/stream.php</jwplayer:streamer>
		</item>
	</channel>
</rss>

加载一个外部的 Media Provider ¶

<config>
   <file>myvideo</file>
   <provider>http://www.mysite.com/providers/myprovider.swf</provider>
</config>

CDNs ¶

http 和 rtmp Providers 可以用几个配置选项来进行定制,从而允许他们连接CDN 和 RTMP 服务器. 这些选项可以使用 flashvars 或一个 xml 配置文件来进行设置

HTTP 流选项 ¶

  • http.startparam
    • 对于BitGravity:MP4就用 starttime ,FLV就用 apstart.
    • 对于 EdgeCast:FLV就用 ec_seek .
    • 对于 Limelight:FLV就用 fs,MP4就用ms.
    • 对于 Lighttpd:FLV 和 MP4都用 start .

MRSS 播放列表的例子:

<rss version="2.0" 
	xmlns:media="http://search.yahoo.com/mrss/" 
	xmlns:jwplayer="http://developer.longtailvideo.com/trac/wiki/FlashFormats">
	<channel>
		<title>Example media RSS playlist</title>
		<item>
			<title>BitGravity Video</title>
			<media:content url="video.flv" />
			<jwplayer:provider>http</jwplayer:provider>
			<jwplayer:http.startparam>apstart</jwplayer:http.startparam>
		</item>
		<item>
			<title>Lighttpd Video</title>
			<media:content url="video.flv" />
			<jwplayer:provider>http</jwplayer:provider>
		</item>
	</channel>
</rss>

RTMP 流选项 ¶

  • rtmp.loadbalance
    • 对于Highwinds: true
    • 对于 VDOX: true

MRSS 播放列表的例子: 

<rss version="2.0" 
	xmlns:media="http://search.yahoo.com/mrss/" 
	xmlns:jwplayer="http://developer.longtailvideo.com/trac/wiki/FlashFormats">
	<channel>
		<title>Example media RSS playlist</title>
		<item>
			<title>Highwinds Video</title>
			<media:content url="http://hwcdn.net/nksdjg/video.flv" />
			<jwplayer:provider>rtmp</jwplayer:provider>
			<jwplayer:rtmp.loadbalance>true</jwplayer:rtmp.loadbalance>
		</item>
		<item>
			<title>VDOX Video</title>
			<media:content url="http://myserver.com/video.flv" />
			<jwplayer:provider>rtmp</jwplayer:provider>
			<jwplayer:rtmp.loadbalance>true</jwplayer:rtmp.loadbalance>
		</item>
	</channel>
</rss>
