用JW Player添加流式视频

  • JW Player 指南

事先请阅读:  配置 JW Player

目的

此指南涵盖了基本的流式视频—它们的工作原理以及如何配置播放器支持它们—并提供了为支持流式视频的播放器和播放列表的配置例子.

介绍

对网络视频而言,有两种视频传递方式: 渐进下载式 和 流式. 此指南主要针对的是流式视频.

如果是渐进下载式,文件会在浏览器缓存中渐进地下载.用此方法播放视频需要对视频文件进行持续的下载 . 这意味着你无法跳转到视频的某个还未下载到的特定部分.

第二种视频传递方式是流式, 这是我们要详细讨论的方式,就是流式视频. 这里, 在客户端和服务端之间有一个持续性的连接,从而建立流; 这个过程没有涉及到下载, 这意味着你可以随时跳转到视频的任何一部分,而不必等待那部分下载完成

典型来讲, 流式视频的代价更加昂贵, 这主要归因于流式服务器的开销. 但另一方面, 流式视频相对渐进下载式视频来说更加可靠、更加安全. 许多视频服务器提供了内置的内容保护机制、加载平衡机制以及带宽切换机制.

内容

  • 流式视频的工作原理
  • 提供流式服务的应用程序
  • 配置 JW Player 播放流式视频
  • 在播放列表中添加一个流式视频

流式视频的工作原理

流式的工作原理非常简单. 每次执行一个寻求操作, 媒体播放器用一系列的变量发送一个请求给服务器—要播放什么文件，以及播放的位置. 服务器就会从那个位置开始提供视频 .

以下是一个简短的例子. 在你开始播放视频后, 不必等待直到视频完全加载 ，你可以随时跳转到任何位置.

提供流式服务的应用程序

一个提供流式服务的应用程序就是一个服务器端软件, 针对音视频流提供增强型文件传递服务. 它允许观看者寻求或跳转到视频的任意位置.

当你嵌入一个渐进下载式视频时, 你需要的仅仅是 file FlashVar. 该 FlashVar 告诉视频播放器视频存放于互联网的什么位置 . 但对于流式视频而言, 需要两个部分的信息—视频的所在路径以及提供视频流的软件所在的路径.

比如:

HTTP

http://www.mydomain.com/path/to/application.php?file=path/to/video.flv

RTMP

rtmp://www.mydomain.com/path/to/application/path/to/video.flv

由于JW Player 无法自动侦测URL中的哪个部分是应用程序,哪个部分是视频, 所以你必须把路径分成 file 和 streamer 两个FlashVar. streamer FlashVar 会告诉播放器提供视频流的应用程序所在的路径:

HTTP

streamer=http://www.mydomain.com/path/to/application.php&file=path/to/video.flv

RTMP

streamer=rtmp://www.mydomain.com/path/to/application&file=path/to/video.flv

注意:文件的路径通常是流式服务器上的相对路径 . 不太可能直接从浏览器读取一个视频文件.

配置 JW Player播放流式视频

配置选项

  • streamer (undefined): rtmp/http 服务器的所在位置. 可以是一个RTMP 应用程序或外部的 PHP/ASP 文件. 更多信息.
  • file (undefined): 媒体文件或播放列表的所在位置.
  • type (JW4) / provider (JW5) (undefined): 决定项目是何种类型的媒体文件,这样播放器就会用对应的provider进行 播放.*

* 注意: JW Player总是试着自动侦测并且播放一个文件. 有时候, 不可能指定一个文件扩展名 (比如你在使用一个 token 安全措施). 这种情况下, 你应该总是设置 type (JW4) / provider (JW5) FlashVar ,从而告诉播放器要如何播放视频.

HTTP

  • http.startparam ('start'): 询问参数,提供给服务器起始位置.

RTMP

  • rtmp.loadbalance (false):如果设置成 true, RTMP 模块 (JW4) 或 MediaProvider (JW5) 会假定 FlashVar 是一个 XML 文件 (目前仅支持 SMIL 格式) ，作用是加载平衡.
  • fcsubscribe (JW4) / rtmp.subscribe (JW5) (false): 如果设置成 true, RTMP 模块 (JW4) 或 MediaProvider (JW5) 会为加载平衡使用一个流订阅机制.

使用流式视频的播放器配置的例子

HTTP

Object / Embed
<object classid='clsid:D27CDB6E-AE6D-11cf-96B8-444553540000' width='300' height='300' id='player1' name='player1'>
    <param name='movie' value='player.swf'>
    <param name='allowfullscreen' value='true'>
    <param name='allowscriptaccess' value='always'>
    <param name='flashvars' value='streamer=http://www.mydomain.com/path/to/application.php&file=path/to/video.flv&type=http'>
    <embed id='player1'
           name='player1'
           src='player.swf'
           width='300'
           height='300'
           allowscriptaccess='always'
           allowfullscreen='true'
           flashvars='streamer=http://www.mydomain.com/path/to/application.php&ile=path/to/video.flv&type=http'
    />
</object>
SWF Object 1.5
<script type="text/javascript">
    var so = new SWFObject('player.swf','flashContent','300','250','9');
    so.addParam('allowfullscreen','true');
    so.addParam('allowscriptaccess','always');
    so.addParam('bgcolor','#FFFFFF');
    so.addParam('flashvars','streamer=http://www.mydomain.com/path/to/application.php&file=path/to/video.flv&type=http');
    so.write('flashbanner');
</script>
SWFObject 2.X
<script type="text/javascript">
    var flashvars = {
            'file':                'path/to/video.flv',
            'streamer':            'http://www.mydomain.com/path/to/application.php',
            'type':                'http'
    };
    var params = {
            'allowfullscreen':     'true',
            'allowscriptaccess':   'always',
            'bgcolor':             '#FFFFFF'
    };
    var attributes = {
            'id':                  'player1',
            'name':                'player1'
    };
    swfobject.embedSWF('player.swf', 'flashContent', '300', '250', '9', 'false', flashvars, params, attributes);
</script>

在播放列表中使用一个流式视频

JW Player 支持流式视频的播放列表. 可以通过以下几种方式实现.

要是所有的视频都来自同一个服务器, 那么你只需定义 streamer 和 file 这两个FlashVars.播放器会自动使用应用程序来流式化播放列表中的所有视频.比如:

streamer=rtmp://www.mydomain.com/application&file=http://www.mydomain.com/path/to/playlist.xml

相应的播放列表 (不包括流式应用程序):

<playlist version="1" xmlns="http://xspf.org/ns/0/">
   <tracklist>
      <track>
         <title>The first video</title>
         <location>path/to/first.flv</location>
      </track>
      <track>
         <title>Second video in the list.</title>
         <location>path/to/second.flv</location>
      </track>
      <track>
         <title>And the final one.</title>
         <location>path/to/third.flv</location>
      </track>
   </tracklist>
</playlist>

如果你的视频来自不同的服务器, 你就必须声明 streamer FlashVar 给每一个播放列表条目 (JW 4.3版本起引进). 你还可以在一个播放列表里混合流式和非流式的视频.比如:

file=http://www.mysite.com/path/to/playlist.xml

相应的播放列表 (包含需要的流式应用程序):

<playlist version="1" xmlns="http://xspf.org/ns/0/">
   <tracklist>
      <track>
         <title>The first video is not streamed</title>
         <location>http://www.mysite.com/path/to/first.flv</location>
      </track>
      <track>
         <title>The second video is streamed</title>
         <location>path/to/second.flv</location>
         <meta rel="streamer">rtmp://www.mydomain.com/application</meta>
      </track>
      <track>
         <title>The last video uses a different streamer</title>
         <location>path/to/third.flv</location>
         <meta rel="streamer">rtmp://www.someotherdomain.com/other_app</meta>
      </track>
   </tracklist>
</playlist>

这些例子使用的是 XSPF 播放列表格式.使用其他播放列表格式的方法见 支持的播放列表.
