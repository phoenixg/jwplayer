JW Player 5 XML-图片 皮肤制作向导 ¶

概述 ¶

JW Player 5版本之前,只能用flash制作皮肤. 但现在, JW Player 5发布了, 设计者们可以用自己喜爱的设计器制作皮肤, 并且保存成位图格式. 允许快速建模，不必编译一个swf文件, 不必具备flash经验也能完成皮肤的制作.

位图格式 ¶

JW Player 5 接受大部分的位图格式，包括:

  • JPG
  • GIF 允许透明
  • PNG (8-Bit) 允许透明
  • PNG (24-Bit) 允许透明和半透明

配置 ¶

XML 文档 ¶

XML Document 存储了皮肤的所有设置.它包含了文本和dock元素的颜色设置, 控制条的边缘和文字大小, 以及皮肤中所有元素的图片所在的路径.

剖析 JW Player 皮肤 ¶

一个播放器的皮肤由自身的设置及其组件所组成. 完整的皮肤 XML 文件, 点击这里.

基本的皮肤 XML 结构 ¶

<?xml version="1.0"?>
<skin version="1.0" name="SkinName" author="http://www.yoursite.com/">
	<settings>
		<setting ... />
		<setting ... />
	</settings>
	<components>
		<component name="controlbar">
			<settings>
				<setting ... />
				<setting ... />
			</setting>
			<element ... />
			<element ... />
			<element ... />
		</component>
		<component name="display">
			<element ... />
			<element ... />
			<element ... />
		</component>
		<component name="dock">
			<element ... />
			<element ... />
			<element ... />
		</component>
		<component name="playlist">
			<element ... />
			<element ... />
			<element ... />
		</component>
	</components>
</skin>

开始制作你的 XML 皮肤文档 ¶

公开的 XML 文档声明表示它是一个 XML 格式的文档, 并且要建立成皮肤. 皮肤元素里面是连个属性: NAME 和 AUTHOR. 

<?xml version="1.0"?>
<skin version="1.0" name="SkinName" author="http://www.yoursite.com/">

你可以用你自己的皮肤名称和网址来代替, author项目也可以填写你的姓名.

皮肤设置 ¶

皮肤的设置部分定义了控制条的文本颜色.

<settings>
	<setting name="backcolor" value="0x000000" />
	<setting name="frontcolor" value="0x000000" />
	<setting name="lightcolor" value="0x000000" />
	<setting name="screencolor" value="0x000000" />
</settings>
  • 背景色backcolor
    dock项目上的层元素使用的颜色
  • 前景色frontcolor
    进度条上的时间代码使用的颜色
  • 亮色lightcolor
  • 屏幕色screencolor
    视频背后的颜色

颜色值用十六进制数表示,就好像在HTML/CSS里一样. 因此, 比如你要制作一个红色的 HTML, 就会赋值成 #FF0000. 而在这个 XML 文档中,就是赋值成 0xFF0000.

XML和图片链接 ¶

图片必须放在皮肤文件夹下的子目录当中. 比如, 控制条的图片应该放在 controlbar 这个子目录下.

组件 ¶

每个组件都是 <components> 元素的子级元素. JW Player 5 皮肤的四个不同的默认组件是:

  • controlbar 控制条组件
  • display 显示组件
  • dock dock组件
  • playlist 播放列表组件

控制条 ¶

控制条组件是 JW Player 皮肤使用的最常用的组件. 它控制了视频的播放, 显示你在视频的什么位置,切换至全屏,以及允许你控制音量. 所有 PNG 文件必须存储在 controlbar 子目录下，以便功能运行正常.

控制条设置 ¶

控制条组件的设置:

  • margin
    播放器在全屏模式下控制条的边缘. 以像素为单位.

背景元素 ¶

背景是一个图像,它会水平拉伸以适应控制条的宽度.

控制条元素 ¶

元素顺序 ¶

  • capLeft
  • playButton/pauseButton
  • prevButton
  • nextButton
  • stopButton
  • divider
  • timeSliderRail
  • timeSliderBuffer
  • timeSliderProgress
  • timeSliderThumb
  • divider 重复使用的元素
  • blankButton
  • divider 重复使用的元素
  • fullscreenButton/normalscreenButton
  • divider 重复使用的元素
  • muteButton/unmuteButton
  • volumeSliderRail
  • volumeSliderBuffer
  • volumeSliderProgress
  • capRight

Caps和 Dividers ¶

控制条中有些元素会允许你在元素间增加空格. 他们没有任何功能,只表示控制条的边缘.

  • capLeft
    控制条皮肤的左边图像
  • capRight
    控制条皮肤的右边图像
  • divider (该元素出现了三次)
    按钮和线段之间的分隔符.

注意: JW Player 5 如果没有 capLeft 和 capRight 元素,那么 XML 文件会加载失败.

按钮 ¶

按钮有两种状态. 它们是 button 和 buttonOver (鼠标移至) 状态 -- 两者应该具备相同的分辨率大小 -- 增加你皮肤的元素交互性.

按钮元素的状态是:

  • playButton, playButtonOver
  • pauseButton, pauseButtonOver
  • prevButton, prevButtonOver
  • nextButton, nextButtonOver
  • stopButton, stopButtonOver
  • fullscreenButton, fullscreenButtonOver
  • normalscreenButton, normalscreenButtonOver
  • muteButton, muteButtonOver
  • unmuteButton, unmuteButtonOver
  • blankButton, blankButtonOver

开关按钮 ¶

根据 JW Player的状态,显示何种按钮. 比如, 视频正在播放时, "playButton" 被 "pauseButton" 元素所替代.

这样的元素对是:

  • playButton / pauseButton
  • fullscreenButton / normalscreenButton
  • muteButton / unmuteButton

时间线 TimeSlider ¶

时间线（TimeSlider）元素是独特的元素,由四个元素互相堆积而成.这四个元素中,有三个元素会根据播放器的剩余空间来自动拉伸至相应的尺寸. 这三个元素是:

  • timeSliderRail
    充当时间线的帧“背景”图像
  • timeSliderBuffer
    文件缓冲指示器
  • timeSliderProgress
    文件进度指示器

要注意,设计元素时请让他们能够水平地扩展为好.timeSliderBuffer 和 timeSliderProgress 元素会动态地增长到总文件长度的某个特定百分比. 最后一个元素, timeSliderThumb 只是一个充当把手的简单图像, 你可以拖动进度条,从而在一个特定时刻进行播放.

音量线 VolumeSlider ¶

音量线（volumeSlider） 元素相对时间线（timeSlider）元素而言很小,但它不会自动扩展. 它的大小随你的设计而定. JW Player 5的皮肤模块会增加 5 pixels 的补白到VolumeSlider的每个边上.

  • volumeSliderRail
    充当音量线的帧“背景”图像
  • volumeSliderBuffer
    总音量.
  • volumeSliderProgress
    目前设置的音量.
  • volumeSliderThumb
    滑动音量的把手，也表示音量级别.

注意: JW Player 5 如果缺少 volumeSliderRail 元素,那么XML 文件会加载失败.

空白按钮 BlankButton ¶

BlankButton 是很多插件用到的额外按钮. 没有 "foreground" （前景）元素 (你也不应该设计一个), 它只有一个背景版本 ( 以及一个"Over" 状态) ,为任何通过插件出现的外来按钮而准备.

显示图标 ¶

显示图标就是你在播放器中间所看到的按钮. 在视频播放以前,你看到的是 "Play" 按钮, 同样当你暂停时看到的也是这个按钮. 当你静音时,你会看到 "Mute" 按钮出现. 显示按钮包含两个部分: 一个针对每个按钮的全局背景元素, 以及一个按钮本身,在背景层上居中显示. 所有图片必须存储在皮肤的 display 子目录下.

注意: bufferIcon 在缓冲时会顺时针旋转.

显示 XML 的语法 ¶

<component name="display">
	<elements>
		<element name="background" src="file.png" />
		<element name="playIcon" src="file.png" />
		<element name="muteIcon" src="file.png" />
		<element name="bufferIcon" src="file.png" />
	</elements>
</component>

Dock 图标 ¶

Dock 图标元素坐落于播放器的右上角,可以显示消息,也可以执行功能. 比如,如果你安装了HD插件, 一旦你切换到了 High Definition 播放为ON, 一个小的HD Dock 图标会出现在播放器的右上角,从而让你知道你在播放 HD视频. Dock 按钮只有一个元素:

  • button
  • buttonOver
    按钮的鼠标移至状态

Dock XML 的语法 ¶

<component name="dock">
	<elements>
		<element name="button" src="file.png" />
		<element name="buttonOver" src="file.png" />
	</elements>
</component>

播放列表playlist ¶

播放列表元素由两个部分组成. Playlist Items（播放列表项目）,以及 Playlist Slider（播放列表下拉线）. 项目图像会水平扩展,且位于视频叙述/缩略图（description/thumbnail）的背后. 下拉线是一个垂直的控制把手 (thumb). 如果元素小于播放列表的高度,那么background 元素会充当播放列表默认的背景.

’

播放列表 XML 的语法 ¶

<component name="playlist">
	<elements>
		<element name="background" src="background.png" />
		<element name="item" src="file.png" />
		<element name="itemOver" src="file.png" />
		<element name="sliderRail" src="file.png" />
		<element name="sliderThumb" src="file.png" />
	</elements>
</component>

为播放器安装一个皮肤 ¶

你必须把皮肤文件解压到网站的目录下, 从而让人可以访问.接着你可以通过 XML 文件引用皮肤文件, 还可以用另一些 flashvars作额外的自定义. 你能使用的Flashvars有 :

Zip 文件结构 ¶

XML文件应该和 ZIP 文件命名一样,且放在对应目录下. 所有属于相应目录的图片文件必须和XML文件里的路径一致.

  • example_skin.xml
  • controlbar (放有图片的文件夹)
  • display (放有图片的文件夹)
  • dock (放有图片的文件夹)
  • playlist (放有图片的文件夹)

Flashvar: 皮肤 ¶

在下面这行中, 你会看见皮肤 flashvar 被加进了播放器. 在这个例子中, 它在exampleskin子目录下寻找 skin.xml 这个文件:

so.addParam('flashvars','skin=path/to/example_skin.zip');

Flashvar: dock ¶

要启用dock, 你可以在参数里增加 Flashvar "dock" 一项. 默认值是 TRUE, 没有必要关掉:

so.addParam('flashvars','skin=path/to/exampleskin/skin.xml&dock=true');

Flashvar: controlbar.position / Flashvar: 控制条.位置 ¶

你可以指定播放器控制条的位置为 TOP 或 BOTTOM . 默认值是 BOTTOM.

so.addParam('flashvars','skin=path/to/exampleskin/skin.xml&controlbar.position=true');

把它们都放在一起 ¶

下面是 JW Player 5嵌入皮肤的完整代码的例子,禁用了Dock,控制条放在了顶部.

<div id="videoDiv">this will be replaced by the JW Player.</div>

<script type="text/javascript">
    var so = new SWFObject('JWPlayer5.swf','mpl','800','400','9');
    so.addParam('flashvars','file=video.flv&skin=example_skin.zip&dock=false&controlbar.position=top');
    so.addParam('allowfullscreen','true');
    so.write('videoDiv');
</script>
