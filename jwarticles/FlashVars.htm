JW Player: Flashvars ¶

以下是播放器接受的所有 flashvars 列表. Flashvars是嵌入HTML代码的变量，用来设置播放器的外观与功能. 所有 flashvars的设置都能通过javascript 和插件进行访问. 更多参见, API 页面.

有些 flashvars 代表了一个状态: 他们是只读的，并且会随着播放器自动更新. 插件和Javascripts可以通过 API进行请求.

注意，你必须在flashvars里替换以下三个字符 ? = &. 转义后的字符见下:

  • ? → %3F
  • = → %3D
  • & → %26

因此, 比如你的 file flashvar 的位置是 getplaylist.php?id=123&type=flv, 这时你必须设置flashvar 为 getplaylist.php%3Fid%3D123%26type%3Dflv.

播放列表属性 ¶

如果要加载播放列表，只需一个属性即可:

  • playlistfile (undefined): XML 播放列表 的位置，会随播放器 开始播放时加载.

可以设置以下flashvars来替代 playlist,用以创建播放列表的单一项目. 他们会设置加载 媒体项目 的各种属性 (比如. 源文件, 预览图 或 标题).

  • author (undefined): 视频的作者, 在播放时或播放列表中显示.
  • date (undefined):媒体的出版日期. 从 4.3版本后支持此功能.
  • description (undefined):文字叙述.
  • duration (0): 文件的时长，以秒为单位.
  • file (undefined):媒体文件或播放列表的位置.
  • image (undefined):预览图的位置;在播放时或播放列表中显示.
  • link (deprecated): 该选项在 5.0版本中不再支持.
  • start (0): 强制播放的起始位置，以秒为单位. 一般视频无法使用(即渐进下载式) , 只能用于流式视频 (HTTP / RTMP).
  • streamer (undefined): rtmp/http 服务器的位置. 可以是一个 RTMP 应用程序或外部的 PHP/ASP文件. 更多信息.
  • tags (undefined): 媒体的关键词. 从 4.3版本后支持此功能
  • title (undefined):视频的标题,在播放时或播放列表中显示.
  • provider (undefined): 决定媒体文件属于何种类型, 从而让播放器选择何种provider 来播放视频. 默认的, 播放器会根据媒体文件的扩展名来识别文件类型. 如果没有合适的扩展名，或者播放器错误地侦测了媒体类型, 也可以手动进行设置. 播放器默认支持以下几种格式:
    • video: 渐进下载式的 FLV / MP4 视频, 以及 AAC音频.
    • sound: 渐进下载式的 MP3 文件.
    • image: JPG/GIF/PNG 图片
    • youtube:Youtube的视频.
    • http: http pseudo-streaming伪流式的 FLV/MP4视频.
    • rtmp: RTMP 服务器播放的 FLV/MP4/MP3 视频 .

除了这些默认的 providers之外, 播放器还支持一些特定的流式服务器或 CDNs. 完整的媒体类型见 支持的文件类型页面.

层 ¶

以下 flashvars 控制着播放器的外观和层.

  • backcolor (undefined): 控制条和播放列表的背景颜色. 默认的皮肤下是白色.
  • controlbar (bottom): 控制条的位置. 可以设置成 bottom, over 和 none.
  • dock (false): 设置为 true ，则会在播放器右上角显示大图标的dock. 从 4.5版本开始支持.
  • frontcolor (undefined): 控制条和播放列表的图标和文本颜色.
  • height (400): 显示的高度，以像素为单位.
  • icons (true):设置为 false ，则会隐藏视频中间的播放图标和缓冲图标. 从 4.2版本开始支持.
  • lightcolor (undefined): 鼠标移至时图标和文本的颜色.
  • logo.file (undefined):外部 jpg, png 或 gif 图片的位置，用以替换水印图片 (Licensed players only 购买版本才有)
  • logo.link (undefined): 点击水印图片时去的链接地址 (Licensed players only购买版本才有)
  • logo.hide (true): 设置为 true时, logo会自动隐藏 (Licensed players only购买版本才有)
  • logo.position (bottom-left): logo显示的位置. 可以设置成 bottom-left, bottom-right, top-left 或 top-right (Licensed players only购买版本才有)
  • playlist (none): 播放列表的位置. 可以设置成 bottom, top, over, right 或 none.
  • playlistsize (180):当 below 它时指高度,当 right 它时指播放列表的宽度.
  • skin (undefined): 皮肤文件 的所在位置,包含播放器的图形. SVN 储藏室包含了一系列的皮肤例子.
  • screencolor (undefined): 显示的背景颜色
  • width (280): 显示的宽度，以像素为单位.

四个颜色 flashvars 必须用十六进制数表示, 就像网页颜色 (比如. FFCC00 浅黄色).不支持 位图皮肤.

行为 ¶

以下 flashvars控制播放器的播放行为.

  • autostart (false): 加载时自动播放视频.
  • bufferlength (1): 开始播放前加载的时长，以秒为单位. 设置低了就会即刻开始播放，设置高了就会缓冲一段时间后再播放.
  • displayclick (deprecated): 5.0版本后不再支持.
  • displaytitle (false): 设置成 true，则会在播放时显示视频标题. (目前还不适用)
  • fullscreen (false): 播放器的全屏状态.这是一个只读的 flashvar, 对于插件来说很有用. 从 4.4版本起支持.
  • item (0):开始播放的 播放列表项目 . 选择一个播放项目，让播放器开始播放.
  • linktarget (deprecated): 5.0后版本不再支持.
  • mute (false): 启动时静音. 该选项可以被用户的 cookie撤销, cookie会存储用户的上次静音状态.
  • playerready (undefined): 播放器完成 设置 后的Javascript 回调函数 .
  • repeat (none): 设置成 list ，则一次播放完整的播放列表, 设置成 always ，则持续播放播放列表，设置成 single，则继续播放播放列表中的某个视频.
  • shuffle (false): 随机选择播放项目.
  • smoothing (true): 设置视频的平滑度, 这样当变成高清晰视频时，你就不会看到方块. 设置成 false 来提升老式计算机/大文件的性能. 从 4.4版本起支持.
  • stretching (uniform): 定义如何调整图片的尺寸. 可以设置成 none (不拉伸), exactfit (不成比例的), uniform (拉伸且带有黑边) or fill (统一, 但完全填充整个播放窗口).
  • volume (90):播放器的起始音量. 可以设置成0 到 100. 用户的上次音量设置保存在cookie当中，因此会撤销该flashvar.

API ¶

以下是有关播放器API的 flashvars:

  • debug (undefined): 设置成 arthropod, console 或 trace ，让播放器记录事件的日志. 从 4.5版本起支持. 4.6版本起会保存在cookie中. 更多信息件 插件文档.
  • plugins (undefined): 这是一个要加载的swf插件的列表，用逗号分隔 (比如. yousearch,viral).每个插件都有一个独特的ID ，且存储在 plugins.longtailvideo.com. 访问 LongTailVideo 扩展部分 查看所有插件.
  • client (deprecated): 该 flashvar已在 5.0停用
  • id (deprecated): 该 flashvar已在 5.0停用
  • version (deprecated): 该 flashvar已在 5.0停用

之前的 flashvars, client, id 和 version 会随 播放器传播的事件，作为参数进行传递.

配置 XML ¶

以上所有 flashvars 都可以写入一个 XML 文件中，接着给播放器提供一个单一的 flashvars:

  • config (undefined): 保存 flashvars 的 XML文件. 对于小巧的嵌入代码或 CDN 流的重定向来说很有用. 下面是一个例子:
<config>
   <image>files/bunny.jpg</image>
   <repeat>true</repeat>
   <backcolor>333333</backcolor>
   <volume>40</volume>
   <playlist>right</playlist>
   <playlist.size>150</playlist.size>
   <controlbar>over</controlbar>
</config>

在嵌入代码中设置的Flashvars 会重写在配置 XML文件中的flashvars. （嵌入的代码优先级更高）
